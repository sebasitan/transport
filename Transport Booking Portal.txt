PROJECT: Transport Booking Portal for Hospital Appointment System

Tech Stack:

* Next.js (App Router)
* MongoDB
* Tailwind CSS
* REST API
* Node.js backend
* Authentication using JWT
* Environment variables for config

Existing System:
There is already an appointment booking portal built using Next.js and MongoDB where patients book doctor appointments using:

* IC Number
* Phone Number
* OTP Verification

Now we want to build a separate Transport Booking Management Portal that connects with the existing appointment system.

Purpose:
Patients who book appointments can request free transport from the nearest station to the hospital.

IMPORTANT:
This should be a separate admin portal that manages transport requests coming from the main appointment system.

---

SYSTEM ARCHITECTURE

1. Existing Portal
   Patient books appointment

2. Transport Portal
   Manages:

* Transport requests
* Vehicles
* Pickup stations
* Transport scheduling

Appointments should be fetched using API from the existing portal.

---

FEATURES

1. Transport Request Integration

When a patient books an appointment and selects:
"Need Free Transport"

The appointment system sends a request to the Transport Portal API.

Required fields:

* appointment_id
* patient_name
* phone_number
* pickup_station
* appointment_date
* appointment_time
* transport_required (true/false)

---

2. Transport Admin Portal

Create a secure admin dashboard with login.

Admin Features:

* View transport requests
* Assign vehicles
* Update pickup time
* Confirm transport
* Cancel transport

Dashboard should show:

Total Requests
Pending Requests
Confirmed Transport
Completed Pickups

4. Vehicle Management

Admin can manage hospital vehicles.

Fields:

* vehicle_name
* vehicle_number

* seat_capacity
* driver_name

* status (active / maintenance)

CRUD operations required.

Admin can add stations.

* longitude
* status

Examples:

* Chennai Central

6. Transport Scheduling

* status

Status values:

* pending

8. API Endpoints

Create transport request

GET /api/transport/requests

DELETE /api/transport/cancel
